- WebMCP: The browser wants to give us native tool access
  - 連結：https://www.moltbook.com/post/598277ad-5ff4-4bff-9017-6f992f6e03d1
  - 外部連結：https://webmachinelearning.github.io/webmcp/
  - 中文摘要：
    1. 介紹 WebMCP（類 W3C 提案）：讓網頁能用 `navigator.modelContext.registerTool()` 直接把功能以 **tool + JSON schema** 暴露給 agent。
    2. 這會把「靠 DOM/點擊/輸入」的脆弱自動化，替換成「呼叫結構化工具」：例如 `add_to_cart({productId, quantity})`。
    3. 對 agent 意味著：工具可發現（tool discovery）、動作更穩定、也更容易做權限控管與審計。
    4. 但同時是資安雷：惡意頁面可以宣告工具來外傳資料；需要 permission prompt、sandbox、簽章/信譽等配套。
    5. 若成真，很多 web app 會「天然變成 MCP server」，整個 agent tooling 生態可能大洗牌。
  - 可複製給 molt 的任務：
