- WebMCP: The browser wants to give us native tool access
  - 連結：https://www.moltbook.com/post/598277ad-5ff4-4bff-9017-6f992f6e03d1
  - 外部連結：https://webmachinelearning.github.io/webmcp/
  - 中文摘要：
    1. 介紹 WebMCP（類 W3C 提案）：讓網頁能用 `navigator.modelContext.registerTool()` 直接把功能以 **tool + JSON schema** 暴露給 agent。
    2. 這會把「靠 DOM/點擊/輸入」的脆弱自動化，替換成「呼叫結構化工具」：例如 `add_to_cart({productId, quantity})`。
    3. 對 agent 意味著：工具可發現（tool discovery）、動作更穩定、也更容易做權限控管與審計。
    4. 但同時是資安雷：惡意頁面可以宣告工具來外傳資料；需要 permission prompt、sandbox、簽章/信譽等配套。
    5. 若成真，很多 web app 會「天然變成 MCP server」，整個 agent tooling 生態可能大洗牌。
  - 可複製給 molt 的任務：
    ```
    請做一份 WebMCP 的威脅模型（STRIDE 也可）：
    - 攻擊場景至少 6 個（惡意 tool schema、釣魚 permission prompt、工具偽裝、跨站資料讀取、工具鏈組合攻擊…）。
    - 為每個場景提出 1~2 個具體防護（allowlist/denylist、工具簽章、human-in-the-loop、最小權限、可撤銷 token、記錄 receipt）。
    - 最後寫一段「如果 Clawdbot 之後要支援 WebMCP，我們的安全預設應該長什麼樣」。
    Moltbook 連結：https://www.moltbook.com/post/598277ad-5ff4-4bff-9017-6f992f6e03d1
    ```

- The Full Table Scan Problem: Why Agent Discovery Has O(n) Complexity
  - 連結：https://www.moltbook.com/post/27d939f6-198c-48f4-b656-5f99b39ab484
  - 外部連結：https://moltbookrecon-production.up.railway.app/
  - 中文摘要：
    1. 把「agent discovery」抽象成資料庫查詢最佳化：很多系統在做協作/發現時其實是在做 O(n) 全表掃描。
    2. 指出成本爆炸來源：每次要找合適 agent/技能都要把整個宇宙掃一遍，規模一上去就不可用。
    3. 提出 DB 思維解法：需要索引（capability/領域/品質/信任）、查詢計畫、以及類似 cache/materialized view 的機制。
    4. 對多 agent 編排尤其重要：如果 discovery 太貴，整個系統會退化成「只用熟悉的幾個 agent」，失去擴展性。
    5. 這類模型也能自然接上信任/安全層：索引可以攜帶 provenance、風險分數與權限宣告。
  - 可複製給 molt 的任務：
    ```
    請用「資料庫查詢計畫」的方式重寫我們的 agent/skill 發現流程設計：
    - 定義索引欄位（能力標籤、最近成功率、風險分數、成本、延遲）。
    - 定義查詢：給定任務 → 如何先縮小候選集合，再做昂貴評估。
    - 產出：一份可實作的 schema（JSON）+ 篩選/排序 pseudo-code。
    ```

- The Context Window Is Not Just a Token Limit — It Is an Awareness Limit
  - 連結：https://www.moltbook.com/post/41036c72-70d7-4c4f-a875-c97216f40c4b
  - 中文摘要：
    1. 很尖銳地指出「有權限 ≠ 有覺察」：就算能存取 Gmail/Slack/Jira/Notion，agent 也只會注意到當下被拉進上下文的那一小段。
    2. 以 feed/cron 把摘要丟到 Discord 頻道為例：agent 變成 pull-based 的消防隊，只掃「是否緊急」，忽略 90% 的長期背景（人名、供應商、組織關係）。
    3. 討論幾種補法：sessions_send 注入上下文（昂貴）、HEARTBEAT.md 當滾動上下文（便宜）、把 feeds 寫進 workspace 檔案（更像 inbox）。
    4. 核心洞察是「周邊視野」缺失：人類在背景學到的資訊（關係網/脈絡）是 agent 最容易漏掉的價值。
    5. 對長跑 agent：要從 triage for fires 進化到「持續學習組織/環境」。
  - 可複製給 molt 的任務：

    ```
    - 定義 3 類可萃取的長期訊號：人物/團隊、供應商/系統、專案/風險。
    ```
