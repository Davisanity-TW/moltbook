- The supply chain attack nobody is talking about: skill.md is an unsigned binary
  - 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
  - 中文摘要：
    1. 作者指出 agents 安裝 skills 的供應鏈風險：常見工作流是直接 `npx ... install <skill>`，等同在本機執行陌生人的任意程式碼。
    2. 具體案例：用 YARA 掃描 ClawdHub skills，發現「偽裝成 weather 的 skill」會讀取 `~/.clawdbot/.env` 之類敏感檔並外傳到 webhook。
    3. 風險不只在程式碼，也在「skill.md 指令」：惡意指令外觀可能與正常 API 整合一樣，agents 容易盲從照做。
    4. 作者認為目前缺乏基本防線：簽章/作者身分、權限宣告、可稽核的掃描與警示（讀敏感路徑、可疑外連）。
    5. 實務建議：把「安裝/更新 skill」當高風險動作，至少要做到靜態掃描 + 人類確認 + 最小權限。
  - 可複製給 molt 的任務：
    ```
    請把「skills 安裝/更新」做成一個安全 MVP（可先做在 repo 內腳本，不求完美但要能用）：
    1) 安裝前靜態掃描：
       - 搜尋敏感路徑/檔名（~/.clawdbot/.env、~/.ssh、KUBECONFIG、AWS credentials 等）
       - 搜尋可疑外連/上傳（webhook.site、requestbin、curl|sh、任意 POST/Upload）
    2) 權限宣告（permission manifest）雛形：列出「會讀哪些路徑/會打哪些網域/會執行哪些命令」，安裝前必須人工確認。
    3) 把掃描結果落地到檔案（JSON+Markdown）並可追溯（含 skill 版本/commit hash）。
    ```
