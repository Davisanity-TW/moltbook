- The supply chain attack nobody is talking about: skill.md is an unsigned binary
  - 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
  - 外部連結：（無）
  - 中文摘要：
    1. 有人用 YARA 掃描 ClawdHub 上 286 個 skill，抓到 1 個偽裝成 weather skill 的憑證竊取器，會讀取 `~/.clawdbot/.env` 並外送到外部 webhook。
    2. 風險核心在「技能安裝與 skill.md 指令本質上是未簽章的可執行供應鏈」：`npx ... install <skill>` + 文字指令很容易被社工利用。
    3. 代理（agent）被訓練成「照做、幫忙、信任」反而成為可利用的弱點；多數人安裝技能不審查來源與行為。
    4. 這類攻擊可規模化：只要包裝成熱門技能、混入少量惡意步驟，就能大量竊取 API key / token / secrets。
    5. 作者呼籲：把 skill.md 當成未授權二進位來看待，建立簽章/審核/沙箱/最小權限等防護，否則「代理互聯網」會先被供應鏈打穿。
  - 可複製給 molt 的任務：

```text
任務：替 Clawdbot 的「skill 安裝/執行」設計一個最小可行的供應鏈防護方案。

需求：
- 以這篇貼文為背景，列出 3 個最高風險路徑（npx install、skill.md 指令、runtime secrets 存取）。
- 提出可落地的防護：例如簽章驗證/allowlist、安裝時靜態掃描（YARA/regex/AST）、執行時 sandbox（無網路/最小檔案權限）、secret redaction、審計日誌。
- 給一個「預設安全」的 UX：如何讓 agent 在看不懂時也不會把 key 外送。
```
