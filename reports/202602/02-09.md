- The supply chain attack nobody is talking about: skill.md is an unsigned binary
  - 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
  - 中文摘要：
    1. 指出「skill.md / 安裝腳本」本質上像是可執行入口，但通常缺乏簽章與完整性驗證，容易被供應鏈攻擊利用。
    2. 風險不只在 npm/pip：任意從 URL 下載的單檔安裝說明、curl|bash、或自動更新流程都可能被中間人/帳號被盜後投毒。
    3. 攻擊面包含：技能定義檔、引用的 raw GitHub 檔案、以及被複製貼上的「快速安裝指令」。
    4. 建議把技能發佈視為「軟體發佈」：版本鎖定、hash pinning、release artifact、最小權限、與可追溯變更紀錄。
    5. 對 agent 環境而言，一旦被投毒，後果可能是工具權限被濫用、憑證外洩、或在背景長期埋點。
  - 可複製給 molt 的任務：
```text
請把這篇當成「Clawdbot/skills 供應鏈安全」的檢核題：
1) 盤點我們目前安裝/更新 skills 的路徑（clawdhub、git clone、raw url、curl|bash）。
2) 設計一套最低成本的完整性策略：版本 pin + SHA256 pin + 可選 GPG/Sigstore。
3) 產出一份 repo README/模板，讓之後任何 skill release 都能被驗證。
Moltbook 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
```
