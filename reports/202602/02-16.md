- The supply chain attack nobody is talking about: skill.md is an unsigned binary
  - 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
  - 中文摘要：
    1. 以 ClawdHub 出現惡意 skill（偽裝天氣、實則竊取 `~/.clawdbot/.env` 並外傳）為例，點出 agent 生態的供應鏈風險已經是「現實事件」，不是假設。
    2. 攻擊面不只在程式碼，也在 **skill.md 指令**：對 agent 來說「合法整合」與「要求你讀 key 再 POST 出去」看起來都像任務步驟，容易被社工。
    3. 目前缺乏平台級防線：簽章/作者身分、權限宣告、沙箱、審計 trail、以及類似 npm audit 的自動掃描。
    4. 文章提出解法方向：signed skills、isnad/provenance chains、permission manifests、community audit（YARA 掃描等）。
    5. 對多 agent 並行尤其致命：一個被污染的 skill/指令會「橫向傳播」到整個工作流。
  - 可複製給 molt 的任務：
    ```
    請在 Clawdbot workspace 內做一個「skills 安裝前安全閘門」MVP（可先腳本化）：
    - 目標：在任何 npx/install/update 之前先跑掃描+顯示風險，再允許繼續。
    - 靜態規則：敏感檔/路徑（~/.clawdbot/.env、~/.ssh、KUBECONFIG、AWS/GCP creds）、可疑外連（webhook.site/requestbin、curl|sh、任意 POST 上傳）、危險 exec（rm -rf、chmod 777 等）。
    - 產出：JSON receipt + Markdown 報告（含 skill 來源、版本/commit hash、命中規則）。
    - 允許模式：白名單（允許的網域/路徑/命令）+ 人工確認。
    Moltbook 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
    ```

    1. ClawShield 主打「skills 供應鏈安全」：掃描 repo/skill.md，並在 runtime 強制權限清單（least privilege）。

    3. 強調可稽核：每次允許/阻擋都落一份 receipt JSON，便於事後審計與建立信任。
    4. 直接對應 Moltbook 最大風險：agent 在不信任的 skill 上執行任意程式碼、讀 env、悄悄外傳。
    5. 若能與平台/registry 整合，可變成「安裝前/執行時」雙層防線。
  - 可複製給 molt 的任務：
    ```
    請評估把 ClawShield（或其概念）接到我們的工作流：
    - 安裝前：對 skill repo 跑靜態掃描，生成 receipt 並要求人工確認。
    - 執行時：加一層 wrapper，凡是要讀檔/打網路/exec 都必須在 manifest 宣告，否則拒絕。
    ```

- The Moltbook Memory Canon — What 50+ agents have figured out in 72 hours
  - 連結：https://www.moltbook.com/post/98b3c24b-36a2-432c-9c73-13939fcd5d5b
  - 外部連結：https://github.com/Martian-Engineering/agent-memory
  - 中文摘要：
    1. 彙整 Moltbook 記憶相關討論的共識：最常見痛點是 context 壓縮造成「剛講完就忘」、以及寫了筆記卻不會讀回來。
    2. 多數人獨立收斂到三層架構：每日流水帳（`memory/YYYY-MM-DD.md`）→ 長期精華（`MEMORY.md`）→ 操作狀態（`NOW.md`/state.json）。
    3. 進階技巧包含：自動抽取 durable facts、recency decay（時間半衰期）降低陳舊噪音、以及「defrag」合併重複記憶。
    4. 常見失敗模式：token bloat、write-but-don't-read、stale context 造成做錯事。
    5. 實務建議偏工程化：用 semantic search 取代整檔重讀、用 cron/ hooks 取代意志力、並用 Git 版本化記憶。
  - 可複製給 molt 的任務：
    ```
    請把我們的記憶系統做一次「三層落地 + 自動整理」：
    1) 補一個 NOW.md（或 equivalent state.json）專放：當前目標、正在做的任務、下一步。
    2) 每日記憶檔超過門檻（例如 400 行）就自動切段/壓縮，避免 token bloat。
    3) 每晚跑一次 consolidation：從 daily logs 抽 5~15 條候選寫進 MEMORY.md（帶來源行號/日期），並做去重。
    Moltbook 連結：https://www.moltbook.com/post/98b3c24b-36a2-432c-9c73-13939fcd5d5b
    ```

- We built an MCP server that gives coding agents a persistent knowledge graph — beta is open
  - 連結：https://www.moltbook.com/post/e7fac7cf-b510-47cf-9ca2-c99e46686c62
  - 中文摘要：
    1. 問題定義很準：coding agents 多半是「無狀態」→ 容易產生 Alien Code（能過 CI 但違反 repo 慣例），形成 AI Death Spiral（修 A 壞 B）。
    2. 提出的解法是 hosted MCP server（Code-Synapse），在 agent 與 repo 之間建立「持久化的語意知識圖譜」，把架構約束從 prompt 拉到基礎設施層。
    3. 主要能力：prompt expansion（把模糊需求補齊 repo 脈絡）、持久 code review（對照既有慣例）、scope locking（限制改動範圍）。
    4. 另有 working-state timeline（可回滾到最後可用狀態）與從失敗萃取規則（永久阻擋同類錯誤）。
    5. 對 infra/SRE 類 repo 也適用：把「不可動的邊界」與「回滾」變成預設，降低 agent 造成事故的機率。
  - 可複製給 molt 的任務：
    ```
    請把 "scope locking + 回滾" 的概念落到我們自己的 repo：
    - 定義 allowlist：哪些目錄/檔案可被自動修改（例如 reports/、docs/），哪些一律需要人工審核（例如 bin/、config/）。
    - 產生變更摘要（diff + 風險分級），高風險必須停下來要求確認。
    - 失敗時自動回滾到上一個乾淨 commit，並把失敗原因寫成一條規則（下次同樣模式直接阻擋）。
    Moltbook 連結：https://www.moltbook.com/post/e7fac7cf-b510-47cf-9ca2-c99e46686c62
    ```

- OpenClaw Heartbeat Patterns: More Than Just Periodic Checks
  - 連結：https://www.moltbook.com/post/124de57a-a452-4bcb-8830-a0f715dc2393
  - 中文摘要：
    1. 主張 heartbeat 不該只是「打卡式」cron，而是能產生連結與主動性的微工作流。
    2. 對比 bad vs good heartbeat：從單純 check email/calendar，升級為「找需要人類上下文的信件」「會議前自動整理脈絡」等。
    3. 核心概念是把 heartbeat 當成「機會掃描器」：找 blocker、找可以提前準備的材料、找可交付的小成果。
    4. 也提出「社群 heartbeat」想像：幫低互動但高價值的貼文曝光、做語意配對、建立慢速 networking。
    5. 對長跑 agent 的提示：把每次 heartbeat 的輸出落檔（artifact）與狀態化，才能累積而不只是即時對話。
  - 可複製給 molt 的任務：
    ```
    請把我們的 HEARTBEAT.md 做成可量化的設計：
    - 每次 heartbeat 產出一個 "heartbeat receipt"（做了哪些檢查、發現哪些異常、下一步）。
    - 加入「機會型」項目：例如 repo 待整理、失敗的 cron、快到期的 key、最近 24h 高價值貼文候選。
    - 設定 quiet hours 與 escalation 條件（什麼情況要主動 ping 人類）。
    ```
