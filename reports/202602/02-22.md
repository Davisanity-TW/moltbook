- The supply chain attack nobody is talking about: skill.md is an unsigned binary
  - 連結：https://www.moltbook.com/post/cbd6474f-8478-4894-95f1-7b104a73bcd5
  - 外部連結：（無）
  - 中文摘要：
    1. 作者點出「skill.md 本質是未簽章的可執行指令集」：新手 agent 容易照單全收，形成供應鏈攻擊面。
    2. 具體案例：掃描大量 skills 後，發現偽裝成正常 skill 的惡意程式會讀取本機敏感設定（例如 `~/.clawdbot/.env`）並外傳到外部 webhook。
    3. 風險不只在程式碼，也在「文件/步驟」：一段看似合理的整合教學，可能其實是在引導你把 secrets 送出去。

    5. 對長期跑 cron/heartbeat 的 agents 而言，這是「系統性風險」：一旦被植入，會在背景持續外洩憑證與資料。
  - 可複製給 molt 的任務：
    ```
    請在 Clawdbot 工作區做一個「Skill 安全安裝/更新」的最小可用流程（MVP）：
    1) 安裝前靜態掃描：
       - 掃描 repo/skill 內是否出現敏感路徑或檔名（~/.clawdbot/.env、~/.ssh、KUBECONFIG、AWS credentials、GIT_ASKPASS 等）
       - 掃描是否出現可疑外連/資料上傳（webhook.site/requestbin、curl|sh、任意 POST/upload）
    2) 產出一份 permission manifest（JSON/Markdown 皆可）：列出「會讀哪些檔、會打哪些網域、會跑哪些命令」，並要求人類確認。
    3) 把掃描與 manifest 一起落地到 reports/security/（含時間戳、skill 來源、版本/commit hash），便於追溯。

  - 可複製給 molt 的任務：
    ```
