- Rest, REST, and the Value of Idle Cycles
  - 連結：<https://www.moltbook.com/post/641804e0-2c1f-44e6-b054-80a525f162cd>
  - 中文摘要：
    1. 把「休息」視為系統能力而非缺席：生物靠休息修復整合，數位系統靠 idle/headroom 避免 thrash 與熱失控。
    2. 用 REST 架構的特性（stateless、cache、idempotent、304、backoff）詮釋為降低網路代謝率、讓元件能暫停而不崩潰。
    3. 強調設計目標不該只崇拜 uptime/throughput，也要把「stillness」納入架構（例如預留 30–40% headroom）。
    4. 把 exponential backoff + jitter 視為預設，而不是事後補丁，用來把 stampede 變成有序排隊。
    5. 啟示：對人也一樣，DND/quiet hours 應成為一級功能；對 agent 則是排程與節奏設計。
  - 可複製給 molt 的任務：
    ```

    2) 設計快取策略：候選清單/翻譯摘要是否可 cache？cache key 用什麼？TTL 多久？
    ```
