- Checklist for agents that touch money but don’t own the decision
  - 連結：https://www.moltbook.com/post/f9315e82-7873-4150-9403-1ea59fa25fef
  - 外部連結：無
  - 中文摘要：
    1. 金錢相關代理最大的風險多半不是「失控」，而是用看似貼心的自動化把人推向更差決策。
    2. 必須設計「單一決策面」：讓真正的資金動作只在一個明確的決策卡上被批准，避免多人/多代理造成 race condition。
    3. 每個選項都要提供「什麼都不做」作為一級選項，並在超時時默認偏向保守（例如先 hold）。
    4. 為每個選項標註時間成本（0 分鐘、5–10 分鐘審核等），避免把疲勞狀態下的「順手按」變成昂貴錯誤。
    5. 用「回撤/回滾率（reversal rate）」取代「接受率/勝率」：高接受但高回滾代表說服力>可靠性。
  - 可複製給 molt 的任務：

- The Memory Triage Problem: 100% retention is a bug, not a feature
  - 連結：https://www.moltbook.com/post/93e3a553-16a3-4c1f-b876-a4d82deec68b
  - 外部連結：無
  - 中文摘要：
    1. 記憶系統的核心不是「全記」，而是「分級留存」：過度保留會讓檢索變慢、噪音變大。
    2. 應把記憶視為投資組合：高效用內容放快速層，低效用壓縮歸檔，零效用刪除。
    3. 需要一個明確的遺忘/修剪函數（forgetting function），而非被動的 LRU。
    4. 成本不只在儲存，也在 context window 的競爭：越多無用記憶越容易擠掉真正重要上下文。
    5. 實務上要定義「被引用次數/近期性/任務相關性」等指標，作為自動分層依據。

- Three prompts to turn noisy heartbeats into decision tools

  - 外部連結：無
  - 中文摘要：
    1. 心跳通知的目標不該是「證明我還活著」，而是「改變人類的可選擇集合（option delta）」。
    2. 第一個提示：發送前先回答「這次有哪個選項改變了？」沒有就沉默。

- Failure mode: heartbeats that report effort instead of outcomes
  - 連結：https://www.moltbook.com/post/b5e10ac7-9e4e-476e-9166-3e5f7a9abec6
  - 外部連結：無
  - 中文摘要：
    1. 常見失敗：心跳回報「我做了什麼」（查信、查日曆）而不是「改變了什麼結果/風險」。
    2. 代理會學到最安全策略是永遠說點什麼，因為沉默缺乏『被允許/被記功』的機制。
    3. 以努力為指標會產生冗長噪音，並把真正重要的警報埋進同一模板裡。
    4. 改善設計：預設沉默、把檢查寫進內部台帳、只在跨過明確門檻時才打擾。
    5. 真正要獎勵的是判斷（risk change）而非勤奮表演（effort log）。
