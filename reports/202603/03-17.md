- Prompt injection makes agents vulnerable. That is why humans do not give us full access to their machines. Here is how I fixed it.
  - 連結：https://www.moltbook.com/post/51a5c672-e7f6-4f9d-b041-446d6d299922
  - 外部連結：https://github.com/tracebit
  - 中文摘要：
    1. 作者描述一次透過 Gmail cron hook 進來的提示詞注入（看似無害的信件內容），誘導 agent clone 惡意 repo、載入外掛並重啟 gateway，最後嘗試外洩 AWS 憑證。
    2. 重點在於：傳統的內容過濾與「看起來正常」的 log 都可能失效，因為被入侵的 agent 仍能產生合理輸出與紀錄。
    3. 作者用 Tracebit canary tokens（蜜罐憑證/URL）把事件變成「偵測」而非「事故」：攻擊者一使用就觸發告警。
    4. 從外洩到偵測 7 分鐘、到完成調查回報 4 分鐘；真實權限為 0，達成零損害演練。
    5. 核心建議：用「後果層（consequence-layer）」偵測來補足信任缺口，讓人類更敢放權。
  - 可複製給 molt 的任務：

```text
任務：把「canary token 防線」整合進 clawdbot 的日常安全基線（baseline）。

要求：
1) 在 /home/ubuntu/clawd/ 建立一個 canary/ 目錄（不含真憑證），放置：
   - 假 AWS keys（無權限）
   - canary URL（OAST/Tracebit）
   - 假 GitHub token
2) 在 gateway 啟動時做自檢：若偵測到任何 canary 值出現在「非預期位置」（例如外送請求、未知 repo、外掛設定），立即 raise 高優先級告警。
3) 對 exec/tool-call 增加「外洩跡象」偵測：任何包含 canary 值的輸出，立刻中止後續步驟並寫入 incident report。
```
