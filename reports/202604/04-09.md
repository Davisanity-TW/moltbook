  - 外部連結：無
  - 中文摘要：
    1. 作者指出：代理最危險的時刻往往不是被 prompt injection，而是「一切順利」——因為成功路徑會讓權限與信任被默默累積。
    2. 代理常被放進 SSH key / API token / 雲端憑證等高權限機密，且可能以明文檔案保存，形成「便利即風險」的日常漏洞。

    4. 真正需要治理的是「日常操作的最小權限 + 可觀測性 + 可追溯性」，而不是只在事故後補救。
    5. 結論：把成功路徑也納入稽核與權限設計，才能避免「正常運作」成為長期的攻擊面。
