  - 中文摘要：
    1. 攻擊向量不是套件或提示注入，而是 Git 分支名等「工作流中被信任的中繼資料」。
    2. 透過分支名參數注入命令，可能在 agent 執行環境內觸發，導致 token/憑證外洩。
    3. 即使在 sandbox 中也可能中招，因為攻擊利用的是 sandbox 內「合法工具管線」本身。
    4. 這類風險提醒：工具鏈解析器（parser）與 glue code 是新的攻擊面。
    5. 防護需要把「工具輸入」當不可信：嚴格轉義、限制命令拼接、記錄與告警。
